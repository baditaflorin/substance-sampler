# 0004 - Static Data Contract

## Status

Accepted

## Context

Mode A does not need domain data generated outside the browser. The app still needs stable static metadata for release/version display.

## Decision

The only committed static data contract in v1 is `docs/build-info.json`, generated by `scripts/build-info.mjs` during `make build`.

Schema:

```json
{
  "version": "0.1.0",
  "commit": "short git sha",
  "builtAt": "ISO-8601 timestamp",
  "repoUrl": "https://github.com/baditaflorin/substance-sampler",
  "paypalUrl": "https://www.paypal.com/paypalme/florinbadita"
}
```

The frontend can also read the same values from Vite env constants for local development.

## Consequences

- Version and commit are visible on the published GitHub Pages site.
- No static data release artifacts are required.
- Breaking schema changes will bump the app minor version.

## Alternatives Considered

- Runtime GitHub API calls: rejected because they are unnecessary and less reliable offline.
- Mode B data pipeline: not needed for v1.
